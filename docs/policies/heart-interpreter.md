# 마음 통역소 정책

## 개요

마음 통역소는 양육자의 육아 고민을 AI 상담사 "아이나"가 듣고, 아이의 기질(TCI)과 양육자 기질의 역동을 분석하여 맞춤 솔루션("마음 처방전")을 제공하는 핵심 기능이다.

경로: `/consult` | 하단 네비 중앙 FAB 버튼

## 전문 용어 제한 (서비스 공통 정책)

사용자에게 노출되는 모든 텍스트에서 NS, HA, RD, P, TCI 등 영문 약어를 사용하지 않는다. 한글 용어(자극추구, 위험회피, 사회적민감성, 인내력)를 사용한다.

## 진입 조건

| 조건 | 상태 |
|------|------|
| 아이 등록 | 필수 (미등록 시 "아이를 먼저 등록해주세요" 안내 → `/settings/child/new`) |
| 아이 기질 검사 완료 (reports 테이블) | 필수 (미완료 시 "기질 검사를 먼저 해주세요" 안내 → `/survey/intro`) |
| 양육자 검사 완료 | 선택 (미완료 시 보편적 양육자 기질로 분석) |
| 로그인 | 필수 (상담 저장을 위해) |
| 구독 또는 트라이얼 활성 | 필수 (만료 시 구독 유도 메시지 표시, CTA 비활성은 아님) |

## 상담 흐름

### 1단계: INPUT (고민 입력)

양육자가 고민 상황을 자유 텍스트로 입력한다.

- **예시 고민 칩**: 입력창 위에 아이 나이·성별에 맞는 예시 고민 5개를 `getRandomExamples(childBirthDate, childGender, 5)`로 생성하여 표시. 탭하면 입력창에 줄바꿈 + 텍스트 append.
  - 이어서 상담(`?sessionId=`) 진입 시 예시 칩 미표시.
- **글자 수 제한**: 최대 500자, 30자 미만이면 CTA 비활성 + "조금만 더 구체적으로 써주시면 정확한 분석이 가능해요" 안내
- **이어서 상담**: `?sessionId=` 쿼리 파라미터로 기존 세션을 이어서 상담 가능. 이전 상담 요약 + 실천 현황 카드가 상단에 표시됨.
- **CTA**: "상담 시작하기" — 30자 미만이면 비활성

### 2단계: DIAGNOSTIC (상황 분석 질문)

AI가 공감 멘트와 함께 상황 분석 질문을 순차적으로 제시한다.

#### 공감 멘트 (empathy)
- 모든 질문 화면에 상단 고정 표시
- 양육자의 상황을 인정하고 공감하되, 기질적 인사이트를 가볍게 포함
- "상담사 아이나" 라벨 뱃지 부착
- 팔로업 질문 진입 시 공감 멘트가 팔로업 사유 설명으로 교체됨 (followUpReason)

#### 질문 생성 규칙
- 정확히 **4개** 질문을 아래 순서로 생성:
  - **질문 1~2**: 객관식(CHOICE) — 아이의 행동을 객관적으로 파악 (상황, 빈도, 맥락)
  - **질문 3**: 객관식(CHOICE) — 부모의 감정 상태나 반응 파악 (비판단적 톤)
  - **질문 4**: 주관식(TEXT) — 구체적 대처 방식이나 경험을 깊이 있게 질문
- 객관식의 마지막 선택지에 `"freeText": true` 가능 — 탭하면 자유 텍스트 입력창이 열림
- 질문 톤은 상담사가 따뜻하게 대화하듯, 문두에 공감적 전제를 포함

#### 팔로업 질문 (선택적 2차 문진)
- 1차 답변 완료 후, AI가 추가 정보 필요 여부를 판단
- 필요 시 심층 질문 1~2개를 추가 생성 (1차 답변 내용을 언급하며 구체화)
- 팔로업 진입 시 `followUpReason`(추가 질문 사유)을 상담사 아이나 멘트로 표시
- 불필요하면 바로 처방전 생성으로 진행
- 팔로업은 최대 1회만 수행

#### 진행률 표시
- "N / M" 텍스트 + 도트 프로그레스 바
- 마지막 2문항에 도달하면 "거의 다 왔어요" 텍스트 표시
- 팔로업 질문 추가 시 전체 수가 동적으로 증가
- 이전 질문으로 돌아가기 버튼 (첫 질문 제외)

### 3단계: RESULT (마음 처방전)

모든 질문 답변 완료 후 처방전을 생성하여 표시한다.

#### 처방전 구조

```json
{
  "interpretation": "아이의 속마음 통역 (5~7줄)",
  "chemistry": "양육자-아이 기질 역동 설명 (4~6줄)",
  "questionAnalysis": [
    { "question": "질문 원문", "answer": "답변 원문", "analysis": "기질 관점 해설 1~2줄" }
  ],
  "magicWord": "상황 반전 대화 스크립트",
  "actionItems": [
    {
      "title": "~할 때, ~하기 (If-Then 형식 권장)",
      "trigger": "구체적 상황/순간 (선택)",
      "action": "30초 이내 행동 (선택)",
      "description": "기질 연결 설명",
      "duration": 5,
      "encouragement": "응원 메시지"
    }
  ],
  "sessionTitle": "고민 요약 제목 15자 이내 (신규 세션만)"
}
```

#### 처방전 항목별 규칙

| 항목 | 목적 | 톤 & 제약 |
|------|------|-----------|
| interpretation | 아이의 행동이 나쁜 의도가 아님을 기질적 욕구로 설명 | **아이의 1인칭 말투로 작성** ("나는 ~해요"), 5~7줄, 문진 답변 근거 포함 |
| chemistry | 양육자-아이 기질 간 역동으로 갈등 원인 설명 | UI 라벨 "우리가 몰랐던 마음의 신호", 양육자를 탓하지 않음, 문진 답변 근거 포함, 4~6줄 |
| questionAnalysis | 각 문진 질문·답변을 기질 관점에서 해설 | 질문 원문(question), 답변 원문(answer), 해설(analysis) 1~2줄 |
| magicWord | 상황을 반전시킬 구체적 대화문 | 따옴표 없이 대화 스크립트만 반환, UI에서 따옴표 표시 |
| actionItems | 정확히 3개의 실천 항목, 양육자가 1개를 선택하여 실천 | Tiny Habits 원칙 (30초 이내), If-Then 형식 권장, 구체적이고 측정 가능, 서로 다른 접근 방식 |

#### 실천 항목(actionItems) 품질 기준
- **Tiny Habits 원칙**: 30초 이내에 할 수 있는 아주 작은 행동. "매일 30분 놀아주기" (X) → "잠들기 전 '오늘 뭐가 재밌었어?' 한마디 묻기" (O)
- **If-Then 권장**: 상황이 명확하면 `trigger` + `action` 필드 포함. 일상 습관형은 If-Then 없이 작성 가능.
- 문진 답변의 구체적 상황에 맞춘 행동 제안. 범용적/추상적 실천 금지.
- `duration`: 3~7일. 짧게 설정하여 성공 경험을 먼저 만듦.

#### 결과 화면 레이아웃
1. 날짜·아이 이름 뱃지
2. 흰색 카드: "아이의 마음 지도" — 고민 원문 인용 + interpretation
3. 흰색 카드: "문진 해설" — questionAnalysis 목록
4. 흰색 카드: "우리가 몰랐던 마음의 신호" — chemistry
5. 틸 카드: "오늘의 한마디" — magicWord (따옴표 감싸서 표시)
6. 실천 항목 선택 영역: "오늘 당장 해볼 수 있는 것" — actionItems 3개 중 1개 선택 (라디오)
   - 선택 시 If-Then 구조 표시 (trigger/action이 있는 경우)
7. CTA: "실천 시작하기" (practice_items에 저장 후 `/practices`로 이동) / "다음에 할게요" (홈으로)
8. 격려 문구: "아이의 마음을 이해하려는 것만으로도 이미 충분히 좋은 부모예요."
9. 의료 관련 안내: AI 상담은 양육 참고 자료이며 의학적 진단·치료·전문 상담을 대체하지 않는다는 점을 명시. 아이의 안전, 건강, 발달에 대한 우려가 있거나 증상이 지속되면 전문기관 도움을 받도록 안내
10. 앱 다운로드 유도 섹션 (푸시 알림 안내)

## LLM 컨텍스트 주입

기질 프로필은 컴포넌트 마운트 시 1회 계산하여, 초기 질문과 처방전 API 모두에 동일하게 전달한다.

### 입력 데이터

| 데이터 | 주입 위치 | 비고 |
|--------|-----------|------|
| 고민 상황 (problem) | initial, prescription | 양육자 원문 그대로 |
| 아이 이름 (childName) | initial, prescription | "N(나이, 성별) 아이의 양육자이고," 형태로 |
| 아이 생년월일 (childBirthDate) | initial, prescription | 나이 계산에 사용 (월령 또는 세) |
| 아이 성별 (childGender) | initial, prescription | male/female |
| 아이 기질 프로필 (childProfile) | initial, prescription | 아래 상세 참조 |
| 양육자 기질 프로필 (parentProfile) | initial, prescription | 아래 상세 참조 |
| 문진 답변 (answers) | prescription | 전체 질문-답변 매핑 |
| 최근 관찰 기록 (recentObservations) | initial, prescription | 최근 5건, 조회 실패 시 빈 배열로 진행 |
| 세션 컨텍스트 (sessionContext) | initial, prescription | 이어서 상담 시만 포함 (이전 상담·실천 기록) |

### 기질 프로필 구조

#### childProfile / parentProfile

```json
{
  "label": "자유로운 탐험가",
  "keywords": ["호기심", "독립심", "당당함"],
  "description": "새로운 것에 거침없이 도전하며 혼자서도 잘 노는 아이",
  "scores": { "NS": 78, "HA": 42, "RD": 55, "P": 60 }
}
```

- 검사 미완료 시 `null` → LLM에게 "검사 데이터 없음"으로 전달

### LLM 프롬프트 주입 포맷

```
**[기질 프로필]**
- 아이 기질 유형: 자유로운 탐험가 (호기심, 독립심, 당당함)
  - 설명: 새로운 것에 거침없이 도전하며 혼자서도 잘 노는 아이
  - 차원별 점수 (0~100): 자극추구=78, 위험회피=42, 사회적민감성=55, 지속성=60
- 양육자 기질 유형: 신중한 관찰자 (관망, 안전중시, 인내심)
  - 설명: 높은 시야에서 멀리 내다보며 아이를 조용히 지켜봄
  - 차원별 점수 (0~100): 자극추구=42, 위험회피=72, 사회적민감성=58, 지속성=65
```

### 관찰 기록 주입 포맷
```
[날짜] 상황: ... → 양육자 행동: ... → 아이 반응: ... (메모: ...)
```

### 관찰 기록 활용 지침
- 이전에 시도한 방법 중 효과적이었던 것은 강화
- 효과 없었던 것은 다른 접근 제안
- "지난번에 ~를 시도하셨는데"와 같이 자연스럽게 언급

### 세션 컨텍스트 (이어서 상담)
이전 상담의 최근 3건 + 실천 기록(로그, 회고)을 프롬프트에 주입한다.
- 이전 상담: 날짜, 고민, 속마음 통역, 마법의 한마디, 문진 해설
- 실천 기록: 항목별 실천 일수, 상태, 회고 내용
- 효과적이었던 방법 강화, 효과 없었던 것은 다른 접근 제안

## API 엔드포인트

| 엔드포인트 | 역할 | 모델 | temperature |
|-----------|------|------|-------------|
| `POST /api/consult/questions/initial` | 공감 멘트 + 기초 질문 정확히 4개 생성 | gpt-4o | 0.7 |
| `POST /api/consult/questions/followup` | 팔로업 필요 여부 판단 + 심층 질문 0~2개 | gpt-4o | 0.7 |
| `POST /api/consult/prescription` | 마음 처방전 생성 | gpt-4o | 0.7 |
| `POST /api/consult/options` | 대처 방식 4지선다 생성 (레거시) | gpt-4o | 0.7 |
| `POST /api/consult/session-context` | 세션 + 상담 + 실천 기록 조회 | - | - |

## 상담 저장

### 세션 기반 구조
- 신규 상담 시 `consultation_sessions` 테이블에 세션 생성 (user_id, child_id, title=sessionTitle)
- 이어서 상담 시 기존 세션의 `updated_at`만 갱신
- 상담 결과는 `consultations` 테이블에 저장 (status: `COMPLETED`)
  - 저장 항목: user_id, child_id, session_id, category("자유 입력"), problem_description, ai_options(질문 목록), user_response(답변), selected_reaction_id("DYNAMIC_FLOW"), ai_prescription(처방전 JSON)
- 실천 항목 선택 시 `practice_items` 테이블에 저장 (session_id, consultation_id, title, description, duration, encouragement)

## 로딩 상태

| 시점 | 표시 |
|------|------|
| 질문 생성 중 / 팔로업 판단 / 처방전 생성 중 | 전체 화면 블러 오버레이 — 스피너 + 단계별 안내 텍스트 + 프로그레스 바 + 상담 내용 카드 |

#### 로딩 오버레이 상세
- INPUT 단계: "아이의 기질을 분석하고 있어요" / "입력하신 내용을 바탕으로 맞춤 질문을 준비 중입니다"
- DIAGNOSTIC 단계: "마음을 번역하고 있어요" / "아이의 마음에 맞는 처방전을 만들고 있어요"
- 양육자가 입력한 고민 내용을 "상담 내용" 카드로 함께 표시하여 맥락 유지

## 기질 데이터 부재 시 동작

| 상태 | childProfile | parentProfile | LLM 동작 |
|------|-------------|---------------|----------|
| 아동 검사만 완료 | 전체 프로필 | `null` | 아이 기질 기반 분석, 양육자는 보편적 기질로 가정 |
| 둘 다 완료 | 전체 프로필 | 전체 프로필 | 기질 역동까지 포함한 완전 분석 |

- 아이 미등록 시 상담 진입 자체가 불가 ("아이를 먼저 등록해주세요" 화면)
- 아이 등록했으나 기질 검사 미완료 시 "기질 검사를 먼저 해주세요" 화면
- 양육자 검사 미완료 시 상담은 가능하나, 양육자 기질 없이 아이 기질 단독 분석으로 진행
